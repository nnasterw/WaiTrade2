# BD08 策略文档 — 2026-06-12（终版）

## 策略定义

**BD08 = BD07 + H4方向锁 + M15结构止损**

BD07 所有 scalping 参数不变（SL=0.40ATR, TimeExit=20, Cooldown=0, Decay=ON）；BD08 将 DTP 放宽到 2.5R/0.40，结构止损持仓会跳过 DTP。
BD08 增加三层叠加改进：

| 层 | 机制 | 效果 |
|----|------|------|
| **H4 方向锁** | H4=BULL→只做多, H4=BEAR→只做空 | 拦截逆势入场，2605 亏损 -87% |
| **BOS Retest** | H1 结构突破→回踩→入场（绕过双扫门控） | 新增顺势信号源 |
| **M15 结构止损** | M15 摆动点 + 2.0ATR 缓冲 → 动态 SL | 取代 DTP，顺势持仓吃满趋势 |

### 核心洞察

2605（2026年5月）是剧烈熊市：XAU 从 4616 暴跌至 4478（$138，17天），然后反弹至 4546（$68）。
传统 scalping（BE=0.5/0.4，紧止损）在趋势市反复被扫出——单边 $138 跌势只吃到零头。
**解决方案**：H4 判方向 → 只跟顺势 → M15 结构 SL 让利润奔跑 → DTP 作为安全网兜底。

## 架构改动

### v2 层（最小改动，仅 3 个文件）

```cpp
// Types.mqh (+1 field)
bool use_structure_sl;   // BD08: 结构止损持仓→跳过DTP/Trail

// PositionManager.mqh (+2 lines)
if(tracks[i].use_structure_sl) continue;  // 跳过DTP/Trail

// EntryEngine.mqh (+1 field)
bool is_bos_retest;  // BOS Retest信号标志
```

### v3 层（主力开发，WaiTrade_OB_SMC.mq5 ~200行）

1. **H4 方向锁**（ExecuteChannelConfirmed, M1 OB 段）：
   - 获取 `g_trend_state_h4`（H4 自适应趋势判定）
   - 逆势 M1 OB 直接 `continue` 跳过
   - 顺势入场设置 `use_structure_sl = true`

2. **BOS Retest 检测**（DetectSwingBreakInline）：
   - H1 swing 突破（2-bar pivot, 48-bar lookback）
   - 突破→延伸（M15 ATR×0.5）→回踩（IsRetesting）
   - 不要求 M1 OB 确认（v10 血训：取消后 2605 +$3）
   - BOS 信号绕过双扫门控

3. **M15 结构止损**（tick 处理，在 SyncPositions 后）：
   - H4 顺势时才激活
   - 每 tick 扫描 M15 最近 30 根 K 线找摆动点
   - SELL 仓：找到最近 M15 swing high → SL 放在其上方 + 2.0ATR
   - BUY 仓：找到最近 M15 swing low → SL 放在其下方 - 2.0ATR
   - 只在 SL 改善（朝盈利方向）时才更新
   - 最小移动阈值：0.3 ATR（防抖）

4. **PosTrack 架构分离**：
   - `use_structure_sl=true` 的持仓：由 EA 层 M15 结构管理 SL
   - `use_structure_sl=false` 的持仓：由 PositionManager DTP/Trail 管理
   - 两条路径互不干扰

### 血训

1. **结构止损不能和 DTP 共存**——DTP 在结构止损激活前就止盈出场，PosTrack 架构分离是根本解决方案
2. **M5 摆动点太噪**——M15 摆动点 + 2.0ATR 缓冲才可靠
3. **BOS 不能加 M1 OB 确认**——加了反而不如不加（v10 血训）
4. **H1 lookback 不能太长**——80-bar 找到 4 月数据误导 5 月判断
5. **Swing 检测找最近（时间），不找最极端（价格）**——最极端会跨月

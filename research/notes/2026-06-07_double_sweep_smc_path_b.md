# 双扫确认 (Double Sweep Confirm) — SMC 路径B 研究笔记

## 背景

SMC分析（`2026-06-07_2605_smc_analysis.md`）提出四条震荡市盈利路径。路径A（Mitigation Entry）已实现并验证。本文记录路径B（双扫确认）的完整实现和验证。

## 核心 SMC 概念

在窄幅震荡区间中，市场通过**双向扫荡流动性池**来确立真正的方向：

```
价格→扫上方LP（buy stops）→ 立即反转 → 扫下方LP（sell stops）→ 双扫完成 → 真正方向出现
```

在双扫完成之前的所有单向扫荡都是**陷阱**——价格会在扫完一侧后立即反转去扫另一侧。当前的 Bounce Entry 在单向扫荡后入场，正好接在陷阱中。

## 实现

### 过滤器：PassDoubleSweepConfirm

```cpp
bool PassDoubleSweepConfirm(const OBZone &zones[], int zone_count, int bar_count)
```

逻辑：
1. 扫描窗口内（默认20 bars）所有 Sweep OB
2. 检查是否存在 BUY 方向 sweep OB（=下方 LP 被扫荡）
3. 检查是否存在 SELL 方向 sweep OB（=上方 LP 被扫荡）
4. **双方向都存在 → 放行**；缺任一方向 → 拦截

### 集成点

在两个入口路径均添加门控：
- **直接入场路径**：ScanSignals 开头
- **EntryEngine 路径**：ExecuteChannelConfirmed 开头
- **HTF EntryEngine 路径**：OnTick 中 HTF 确认块

### 激活机制

`OnlyDefensive=true`：仅在 `IsAdaptiveNoiseGateDefensive()`（权益回撤>3%）时启用。

### 自适应特性

双扫确认是**天然的市场体制自适应机制**——不需要检测趋势/震荡，过滤器自身在不同市场中表现不同：

1. **趋势市场（2505）**：价格持续推过各级别 → 双方向 LP 都被扫荡 → 过滤器放行 → 捕捉趋势利润
2. **震荡市场（2605）**：价格在区间内来回 → 单向扫荡后立即反转 → 双扫未完成 → 过滤器拦截 → 避免陷阱
3. **混合市场（2604）**：选择性放行双扫完成后的高质量交易 → 盈亏接近平衡

## 与 Path A (Mitigation Entry) 对比

| 维度 | Path A (Mitigation) | Path B (双扫确认) |
|:---|:---|:---|
| 方法 | 改变**入场时机**（等价格离开再回归） | 改变**入场资格**（需双方向扫荡） |
| 代码量 | ~150行，状态机改动 | ~50行，过滤器模式 |
| 风险 | 复杂状态机，tick振荡bug | 简单过滤器，无状态 |
| 2605效果 | -$35.17（vs S2: -$9.36） | **-$14.11（vs S2: +$8.70）** |
| 2505效果 | +$282,010（SWP基础，非可比） | **+$6,195（vs S2: +$3,069）** |
| 可靠性 | 中（已修复tick振荡bug） | **高** |

## 结论

**Path B（双扫确认）是 SMC 四条路径中最有效的单一改进。** 它是首个在全部三个测试月份均超越 S2 原版基线的配置。

Path A+B 组合过度过滤（2605仅8笔交易），不如单独 Path B。

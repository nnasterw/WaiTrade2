## 核心规律

### 1. 季节性: 7月系统性亏损
```
Jul 2024: -40%  ⚠️
Jul 2025: -30%  ⚠️
```
两年7月都亏。夏季低波动 → SL距离窄 → BE锁过早截断利润。M5在低波动环境里赚不到钱。

### 2. 月份聚集效应
```
好月聚集区: 2-6月 (春), 9-12月 (秋冬)
坏月孤立点: 7月, 1月, 10月(仅2024)
```

### 3. Jan25悖论: 69% WR + PF 1.66 却亏37%
48笔交易, 胜率高但每笔赢太少(+0.39R)而亏太完整(-1R)。M5最小手数0.05 × 宽SL = 一两笔亏损就能吞掉大量盈利。

### 4. 好月的共同特征
- 趋势明确且持续 (H1/H4方向一致)
- market_close退出占比高 (DTP跑出大波段)
- 交易集中在1-3个交易日
- PF通常>1.3

### 5. 坏月的共同特征  
- 震荡/反转市 (趋势方向频繁切换)
- -1R止损集群 (某天3-5笔连续止损)
- 交易分散在多天 (每天都开但每天都亏)
- PF<1

## 改进方向

### A. 7月降风险 (最确定)
两年来7月都亏。用月份上下文过滤:
- `context_filter1_months: "7"` → 7月激活
- `context_filter1_mult: 0.3` → 仓位降至30%
- 预期: 7月亏损从-35%降到-10%

### B. 月暖机 (防Jan25悖论)
- `monthly_warmup_profit_pct: 3.0` → 月盈3%才解锁
- `monthly_warmup_pos_mult: 0.4` → 解锁前40%仓位
- 3%=$6, 0.4x仓位每赢≈$0.45 → 需要13笔赢, 在坏月自然达不到→变相保护
- 好月: 前几笔就盈利→快速解锁→不影响暴赚

### C. OB冷却减速
- `ob_reentry_cooldown_min: 3` → 同OB 3分钟冷却
- 减速集群交易, 在震荡市减少"追涨杀跌"循环
- Jan25的48笔→预计降到30-35笔

### 风险
- 7月降风险可能错过某年7月的好行情
- 月暖机可能延迟好月的爆发(但3%门槛低, 影响小)
- OB冷却可能减少密集趋势日的交易量

## 3. 核心发现

1. **DTP partial 不适合直接迁移到加密**  
   对 BTC 完全无影响，对 ETH 只是把交易数从 43 提到 46，但净利略降。说明加密当前不是“DTP首次回撤没吃满”的问题。

2. **BTC 需要更大 SL/更晚 BE，ETH 不适合**  
   `SL2.0ATR` 让 BTC 180天 PF 到 2.01、净利 +$195.64、DD 降到 26.03%；但 ETH 直接转亏。`BE2.5/1.25` 也只利好 BTC。

3. **OB6x 是唯一双品种同时改善的方向**  
   `min_ob_spread_mult=6.0`：BTC 净利 +$82.42、PF 1.40；ETH 净利 +$34.42、PF 1.77，且 ETH DD 从 14.35% 降到 11.91%。缺点是 ETH 交易数降到 20，样本偏少。

4. **BTC/ETH 应拆分参数**  
   BTC 最优方向偏“更宽 SL / 更晚保护 / 更严 bounce”；ETH 更怕过宽 SL 和过晚 BE，适合更严 OB 过滤而不是放大 SL。

## 4. 当前候选

### 组合候选：v99j2

基于 v99j1，只改：

```yaml
min_ob_spread_mult: 6.0
```

理由：BTC/ETH 双品种都改善，组合净利 +$89.92 → +$116.84，组合 gross PF 约 1.47，最大单品种 DD 不恶化。

### BTC 专属候选

```yaml
sl_buffer_atr: 2.0
```

理由：BTC 180天净利 +$195.64、PF 2.01、DD 26.03%，显著强于基准。但 ETH 失效，不能作为统一 v99j2。

### 次级 BTC 候选

- `breakeven_r=2.5`, `breakeven_lock_r=1.25`
- `bounce_pct=0.50`
- `timeout_min=1080`

均改善 BTC，但对 ETH 不佳。

### 365天结论

1. **统一 v99j2 候选从 OB6 改为 `bounce_pct=0.50`**。  
   180天 OB6 双品种都改善，但 365天只提升 BTC，ETH 净利从 +$122.32 降到 +$72.01，DD 从 21.10% 升到 26.96%。反而 `bounce50` 在 365天里组合净利从 +$276.51 提到 +$314.76，组合PF从 1.35 提到约 1.41，最大单品种DD基本不变。

2. **BTC 专属最强仍是 `sl_buffer_atr=2.0`**。  
   BTC 365天 +$396.67 / PF 1.86，显著强于基准；但 ETH 只剩 +$14.32 / PF 1.05，不能统一使用。若允许品种拆分，应设 BTC 专属版本。

3. **BE2.5/timeout1080 不适合作为统一参数**。  
   它们对 ETH 或整体组合伤害明显。

## 6. 推荐方案

### 统一版 v99j2

基于 v99j1，只改：

```yaml
bounce_pct: 0.50
```

适合 BTC+ETH 同一策略部署：365天组合净利 +$314.76，相对 v99j1 +$38.25；组合PF约 1.41，最大 DD 基本不变。

### 分品种版

- BTC: `sl_buffer_atr: 2.0`
- ETH: 暂保留 v99j1，或继续围绕 `bounce_pct=0.50` / OB过滤做更细实验。

分品种版本收益潜力更大，但 live 部署和参数管理复杂度更高。当前最稳妥是先固化统一 `v99j2=bounce50`，另保留 BTC 专属研究线。

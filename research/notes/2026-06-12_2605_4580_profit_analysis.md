### 核心合成发现

**发现1：4580 OB区间是绞肉机，而非印钞机。**

$4570-4580区间内27笔交易，WR=30%，净亏-$87。市场普遍认为"4580是强压力位，触及做空赚大钱"——这是错的。钱不在OB区间内的反弹交易，而在OB区间突破后的追单。

**发现2：Entry Timing 错误是根因，不是参数问题。**

Decay将180T降到37T，但保留的37T仍然使用完全错误的入口逻辑（Bounce Entry在震荡市中假设"弹回=趋势延续"，实际弹回是反向陷阱）。所有之前的参数优化（RangeATR、Neutral、OB Cooldown）都是在错误的入口逻辑上微调，天花板不是参数，是Entry Timing。

**发现3：现有功能已可实现部分修复，无需全新模块。**

- P1 (H4趋势强制对齐): 2605 6d数据中$39→$103 (+163%)
- P2 (延长持仓时限): 2605短线亏损(持仓<60s WR~26%) → 5min-2h唯一盈利区间
- BOS Retest入口: 可直接捕获4580突破后追单

### 最高评分改进建议

**BOS Retest入口（突破回踩入场）立即实施**。优先级最高，因为：
1. 直接针对4580核心问题——钱在突破后而非突破前

2605延续了2026年上半年的亏损趋势，但亏损幅度最小（绝对值）。这是低交易量的副作用——decay太过激进，以至于根本没给策略机会亏大钱。

### 2.2 Decay 为什么抑制了入场

BD07/S2的decay机制通过价格状态门控大幅降低交易数：

```
2505 (牛市, 583笔):  价格持续偏离 → decay不激活 → 高频入场
2605 (震荡, 37笔):  价格反复回撤 → decay持续激活 → 入场数-94%
```

2605的avg_W=$1.01 / avg_L=$1.41——不是"盈利太小被截断"，而是SL=65美分+BE=26美分把潜在的$20-60盈利交易切成了碎末。

### 2.3 哪些过滤器屏蔽了潜在盈利交易

| 过滤器 | 2605拦截率 | 是否伤害了盈利交易 |
|--------|:--------:|:---:|
| Decay (价格状态) | ~94% | 是——同时拦截了盈利和亏损，但盈利交易因SL太小即使入场也难赢 |
| OB Cooldown | ~10% | 是——4580区域反复扫荡，冷却期正好错过突破后追单 |
| Bounce 确认 (22%回调) | ~30% | 是——震荡市弹回=陷阱，弹回确认是错误假设 |
| EntryDepth=67% | ~15% | 部分是——深度推入OB在趋势市中增强确认，震荡市中加深陷阱 |

核心矛盾：**震荡市中OB越深，被反向扫的概率越大。**

### 4.2 错失原因深度分析

**错失1 (BOS Bear @4580):** 价格在5月3-4日反复触及4570-4590区间。每次触及→弹回→Bounce Entry做多→被扫SL。当真正的跌破来临时，策略已处于OB冷却期（reentry cooldown）或已放弃该OB。

**错失2 (BOS Bull @4600突破):** 价格在5月5-6日突破4600。Bounce Entry要求在OB内反弹确认后再入场，但突破后不存在"弹回OB上方再确认"——价格直接拉升。

**错失3 (Sell @4720):** 发生了38笔交易并盈利$59，说明并非完全错失。但每笔avg_W=$1.55，被BE+TimeExit切碎。如果使用MTF宽SL/H1 ATR SL，avg_W可提升至$3-5。

**错失4 (Buy @4480):** 底部反转发生在5月底，H4结构由BEAR转向BULL。Bounce Entry在底部反转中效果较好，但仓位被decay限制在最低。

### 5.2 顶级推荐详情

#### P0: BOS Retest入口（突破回踩）

**基本原理：**

```
H1收盘突破swing high/low → BOS确认 →
  等待价格回踩突破位（阻力变支撑/支撑变阻力） →
    M1 OB在回踩位反弹确认 → 入场 → H1 ATR宽SL → 中性目标
```

**具体参数设计：**

```
bos_retest_entry: true
bos_retest_sl_buffer: 0.5         # SL=0.5×H1_ATR (~$8-12)
bos_retest_tolerance: 0.3         # 回踩容忍度(ATR倍数)
bos_retest_max_bars: 480          # BOS信号有效期(M1 bars=8小时)
bos_retest_weight: 1.5            # 仓位权重(×InpMTFH1Weight)
```

**4580适用场景：**

```
H1 Swing Low @4580 → 收盘价跌破4580 → BOS Bear →
  价格回踩4580-4590区域 →
    M1 OB确认 → Sell入场 → SL=$12(4580+12=4592) →
      目标1: 4550(前低), 目标2: 4520(结构目标)
```

**预期贡献：** 仅4580区域即可捕获1-2笔额外盈利交易，每笔$30-60。全月贡献$50-150。

**优势：** 代码已在v3架构中实现（`BOSEntry`模块），仅需参数调优和验证。

#### P1: H1 ATR SL替代M1 ATR SL

**根因修复：** M1 ATR=$1.63 → SL=0.4×$1.63=$0.65（被点差吃掉25%）。H1 ATR=$25 → 0.5×$25=$12.50（给OB交易足够的容错空间）。

**实现：** 新增`InpSLUseH1ATR`（默认false），启用后SL计算使用`g_state.atr_1h`替代`g_state.atr_value`。

**风险：** 2505短线scalper交易的SL从$0.65变为$12.50，单笔亏损放大19倍。需要同时收紧其他参数（更高bounce_pct、更严格的质量过滤）来抵消。

**折中方案：** 仅M1（非MTF）交易启用H1 ATR SL，因为MTF本身已使用更宽的SL逻辑。

#### P1: H4趋势强制对齐

**实现：** `InpMTFBlockCounterTrend=true`。H4=BULL时禁Sell，H4=BEAR时禁Buy。已验证2605 6d提升163%（$39→$103）。

**调整：** 不拦截M1交易（留给高频scalper），仅拦截MTF交易。M1交易使用更宽松的过滤条件。

#### P2: 延长持仓时限

**现状：** `TimeExitBars=40`（M1约40分钟），2605短线持仓=绞肉机。

**调整：** `TimeExitBars=120`（M1约2小时）。已验证2505 30d提升41%（$1,847→$2,610）。

**配合：** 与H4自适应协同，CHOP时TimeExit=120，BULL/BEAR时TimeExit=80。

#### P2: OB Freshness v2

**问题：** 4580区域被反复扫荡，每次触及形成OB→入场→亏损→OB仍然有效→再次入场→再次亏损。

**方案：** OB寿命追踪（首次形成到现在的时间），超过ACTIVE_WINDOW（如48小时）的OB降低仓位权重。被缓解2次以上的OB标记为STALE，不再交易。

**实现：** `ObZone.lifetime_bars` 跨bar持久化追踪，不是简单的zone重建。

### 7.2 2505退化风险

最核心的风险是任何改进都会伤害2505的趋势市表现：

| 改进 | 退化机制 | 退化幅度估计 |
|------|---------|:----------:|
| BOS Retest | 新增入口可能在没有BOS的趋势回撤中抢了Bounce Entry的交易 | ~5% |
| H4趋势对齐 | 2505 H4=BULL时禁Sell，利润下降 | ~10% |
| H1 ATR SL | 宽SL让亏损单亏损扩大 | ~15-20% |
| 延长持仓 | 趋势市中本应快速离场的交易持有过久 | ~5% |

**关键洞察：2505的高利润($3,126)主要是因为交易量大(583笔)，每个微利累积。如果改进减少50%交易量但提高50%质量，双月合计可能更好。**

### 7.3 缓解策略

| 风险 | 缓解策略 |

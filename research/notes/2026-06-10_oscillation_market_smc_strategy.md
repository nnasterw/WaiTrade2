## 二、SMC在震荡中的盈利路径

### 2.1 核心策略反转

要让SMC在震荡中盈利，必须反转核心假设：

```
趋势SMC: 扫荡流动性 → 方向确定 → 跟随方向
震荡SMC: 扫荡流动性 → 预期反转 → 反方向入场
```

### 2.2 五种可行策略

#### 策略A：FVG Fade（公允价值缺口回补）

**原理**：价格冲过区域时留下FVG → 震荡市必回补

```
做空信号: 价格sweep上方流动性 + 留下bullish FVG → 做空到FVG补齐
做多信号: 价格sweep下方流动性 + 留下bearish FVG → 做多到FVG补齐
```

**优缺点**：
- ✅ 已实现（FVGDetector.mqh + CheckFVGEntry）
- ✅ 2602验证 PF≈1，-$8（vs base -$247）
- ❌ 方向敏感——FVG在趋势市中不补，可能造成亏损

#### 策略B：Sweep→Reverse（扫荡反转）

**原理**：sweep到达range边界时预期反转而非延续

```
Range[上沿,下沿]，价格sweep上沿 → 做空，TP=下沿
Range[上沿,下沿]，价格sweep下沿 → 做多，TP=上沿
```

**优缺点**：
- ✅ 直接解决方向问题
- ✅ range目标天然是震荡TP
- ❌ 需要精确的range检测
- ❌ 如果市场打破range，扛大亏

#### 策略C：Structure Breakout Filter（结构突破过滤）

**原理**：区分真突破和假扫荡

```
假扫荡: sweep到range外 + 影线 + 收盘在range内 → 反转交易
真突破: sweep到range外 + 实体收盘 + 连续N根同向 → 趋势交易
```

#### 策略D：HTF Gate + Auto-Fade

**原理**：用HTF判断市场状态，震荡态自动反转

```
M15 Net Push ≈ 0 → 标记为震荡态
震荡态: 所有 sweep+OB 交易自动反转方向，仓位减半
趋势态: 正常 sweep+OB 跟随交易
```

**优缺点**：
- ✅ 方向自动匹配市场状态
- ✅ 已有CfgEnableHTFNetPushFilter基础设施
- ❌ HTF信号滞后（至少1-3根K线）

#### 策略E：Swing Target TP

**原理**：震荡市DTP是亏损源，range对侧才是TP

```
正常DTP: sweep → OB → DTP 1.0R → 被反转吃掉
Range TP: sweep → OB → TP = range对侧 → 到价止盈
```

**优缺点**：
- ✅ 已有SwingCapture TP逻辑
- ✅ 简单直接，不新增信号

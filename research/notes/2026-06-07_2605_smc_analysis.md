# 2605 纯SMC分析：震荡月亏损根本原因与Mitigation Entry方案

## 背景

脱离已有参数优化框架，从SMC/ICT第一性原理重新审视2605（2026年5月，亏损月）的市场特征和盈利路径。

## 核心发现：入口时机完全错误

**当前策略在2605的问题不是参数，是Entry Timing在所有时间点上选错了方向。**

### 市场体制诊断

2605 XAUUSD处于**窄幅震荡吸筹期**（ICT Accumulation Phase）：

| 指标 | 2505（趋势月） | 2605（震荡月） | 含义 |
|:---|---:|---:|:---|
| avg_W | $22.98 | **$1.07** | 价格偏移骤降21倍 |
| DTP触发率 | 21.3% | **5.8%** | 目标到不了位 |
| SL率 | 53.6% | **83.3%** | 止损被反复扫荡 |
| 持仓<60s亏损 | ~30% | **~74%** | 入场即逆向 |

本质：市场在窄幅区间内双向扫荡LP（流动性池），每次价格推到一个区间极值就立即反转去另一侧。

### 当前Bounce Entry为什么在震荡月亏

```
价格进入OB
  → EntryDepth=67%（推入OB最深处）
  → BouncePct=22%（等弹回确认）
  → 入场买入
  → 价格立即反转去区间另一侧
  → 扫SL亏钱 ← 83%的概率
```

关键：Bounce Entry假设"价格弹回后会继续走趋势方向"，这仅在趋势月成立。在震荡月，弹回是陷阱——价格弹回OB上方（入场信号）后，下一步是去扫下方LP。

数据验证：
- `bounce_sec < 5s`: 2605 WR=36.7%（越快越亏）
- `bounce_sec > 60s`: 2605 WR=54.2%（等久一点反而好）

### SMC核心概念：Mitigation Entry（消解入场）

ICT最关键的震荡市概念：**价格扫荡LP后会回归被扫荡的区间进行消解。消解回归才是真正的入场时机，不是扫荡弹回时。**

```
当前(错误方向)：
  扫荡发生 → 弹回 → 入场 → 被反向扫

SMC(正确方向)：
  扫荡发生 → 价格下杀完成 → 价格回归消解 → 入场 → 捕捉消解后的方向
```

### 四路径对比

| 路径 | 预期2605收益 | 复杂度 | 代码量 | 说明 |
|:---|---:|:---|---:|:---|
| A: Mitigation Entry | +$50-150 | 中 | ~150行 | **最高优先级** |
| B: 反方向双扫确认 | +$30-80 | 低 | ~80行 | 新增过滤层 |
| C: FVG回补交易 | +$20-60 | 中 | ~200行 | 全新原语 |
| D: 入口逻辑重构(A+B) | +$100+ | 高 | ~400行 | 兼容保留Bounce |

### 为什么之前参数优化没用

- Decay把180T降到37T，但剩的37T仍然用错的入口逻辑，-65→-14
- RangeATR、Neutral、OB Cooldown都是在错误的入口逻辑上微调
- **天花板不是参数，是Entry Timing**

### 下一步

实现Mitigation Entry模式：在EntryMonitor新增状态机
```
PHASE_WAITING_TOUCH → PHASE_WAITING_BOUNCE（现有）
PHASE_WAITING_TOUCH → PHASE_WAITING_SWEEP → PHASE_WAITING_MITIGATION → ENTER（新增）
```

两种模式共存，通过参数 `InpEnableMitigationEntry` (默认false) 控制。
在震荡体制通过ATRRegime自动切换，或防御态时启用。

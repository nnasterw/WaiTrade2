# WFYS 验收标准 v2.0

日期: 2026-07-08
适用范围: WaiTrade2 全部研究版策略
用途: 统一策略验收口径, 解决"只看总利润"或"只看盈利月数"导致的失真。
基于 v1.0 (2026-06-25) 升级, 强化双主轴测试方式, 新增"周均 2 单"硬指标。

---

## 1. 设计目标

WFYS = `WF验收`。

它不是单一利润榜, 而是三件事同时成立的验收模板:

1. 策略必须能赚钱, 不能只有平滑没有利润奔跑。
2. 策略必须稳定, 不能只靠少数爆发月生存。
3. 策略必须具备可研究版 Live 的风险质量, 不能用大回撤换净利。
4. **新增**: 策略必须有足够的交易频次, 保证 Live 资金利用率 (周均 ≥ 2 单)。

WFYS 固定采用三段式输出:

1. `硬门槛`
2. `100分评分`
3. `分级结论`

并附带两个固定诊断字段:

1. `硬失败原因`
2. `主要拖累项`

自动化入口:

- `scripts/wfys_score.py`

## 2. 校准样本

WFYS 的阈值不是拍脑袋得出, 当前先用 XAU 两个已知样本做校准:

| 样本 | 角色 | 24独立月累计 | 盈利月数 | 720d净利 | PF | 结论 |
|---|---|---:|---:|---:|---:|---|
| `v12xau1` | 利润强样本 | 41976.80 | 19/24 | 71700.12 | 2.00 | 爆发力强, 但稳定性和集中度不能直接毕业 |
| `v12xau2` | 稳定样本 | 5572.17 | 22/24 | 8065.77 | 1.57 | 月度更平, 但趋势利润奔跑能力不足 |

结论:

- `v12xau1` 与 `v12xau2` 只作为阈值校准参考样本, 不构成验收通过样本。
- 通过策略必须同时吸收二者单侧优点, 并修正二者单侧缺陷。
- **v2.0 新增**: 校准样本需补"周均 2 单"维度, BTC EA 实测周均 1 单左右时, 应警觉频次不足风险。

## 3. Core 规则

### 3.1 双主轴 (v2.0 重定义)

WFYS 固定同时检查两类回测, **测试方式严格分离**:

1. **`720d 连续回测`** (--continuous-report)
   - 单次连续回测 720 天
   - 看连续盈利能力: 胜率、盈亏、周单数、余额曲线
   - 看真实资金路径: 回撤、恢复能力、风险调整收益
   - `720d` 风险项否决权更高半级

2. **`24 独立月回测`** (--monthly-csv)
   - **24 次独立 30 天回测** (每月 1 次, 从 2024-07 到 2026-06)
   - 每次从月初开始, 不与上月连续
   - 看月稳定性、月份适应性、利润集中度
   - 关联硬门槛: 至少 22 月盈利

规则:

- 二者都必须通过。
- `720d` 风险项否决权更高半级。
- `24 独立月` 负责看稳定性、月份适应性、利润集中度。
- `24独立月整体能力`、`720d整体能力` 统一回到初始资金口径, 便于跨策略比较。

### 3.2 口径统一

收益率口径固定如下:

- 月度分类指标: `当月净利润 / 月初余额` (24 独立月)
- 跨月总收益指标: `总净利 / 初始资金` (720d 连续)
- 连续风险指标: 使用 MT5 连续权益曲线指标 (720d)

其中:

- `强利润月`、`大趋势月`、`大亏月` 全部按月初余额口径定义
- `24独立月整体能力`、`720d整体能力` 统一回到初始资金口径, 便于跨策略比较。

### 3.3 输出优先级

WFYS 的判定顺序固定为:

1. 先过 `硬门槛`
2. 再算 `100分评分`
3. 最后给出 `分级结论`

高分不能豁免硬门槛。

## 4. 硬门槛 (v2.0 完整列表: 19 项)

### 4.1 24独立月稳定性门槛 (v2.0 强化)

| 项目 | 硬门槛 | 说明 |
|---|---:|---|
| 24月盈利月数 | `>= 22/24` | v1.0 是 21/24, v2.0 严格到 22/24 (周均单数要求隐含) |
| 亏损月数量 | `<= 2` | v1.0 是 3, v2.0 严格到 2 |
| 大亏月 | `= 0` | 任意出现直接失败 |
| 大亏月定义 | `月亏损 / 月初余额 > 20%` | 固定为动态权益口径 |
| 单月亏损绝对保护 | `任一亏损月亏损 / 初始资金 <= 100%` | 防止前期暴赚掩盖后期失控 |
| **周均 2 单 (v2.0 新增)** | `720d 总单数 / 103 周 >= 2.0` | 防止低频次策略 |

### 4.2 24独立月趋势利润门槛

| 项目 | 硬门槛 | 优秀线 |
|---|---:|---:|
| 强利润月定义 | `月盈利 > 25%` | - |
| 大趋势月定义 | `月盈利 > 55%` | - |
| 强利润月数量 | `>= 3` | `>= 5` |
| 大趋势月数量 | `>= 1` | `>= 2` |

### 4.3 利润集中度门槛

| 项目 | 硬门槛 | 优秀线 |
|---|---:|---:|
| `Top 3` 利润月 / 24月总利润 | `<= 60%` | `<= 50%` |
| `Top 5` 利润月 / 24月总利润 | `<= 75%` | `<= 65%` |

说明: 该项用于防止策略变成"只靠两三个月吃饭"的彩票结构。

### 4.4 720d 风险质量门槛 (v2.0 强化)

| 项目 | 硬门槛 | 优秀线 |
|---|---:|---:|
| Relative Max Drawdown | `<= 25%` | `<= 18%` |
| Recovery Factor | `>= 3.0` | `>= 5.0` |
| Profit Factor | `>= 1.75` | `>= 2.10` |
| **720d 周均单数 (v2.0 新增)** | `总单数 / 103 周 >= 2.0` | `>= 2.5` |
| **720d 胜率 (v2.0 新增)** | `>= 35%` | `>= 40%` |
| **720d 盈亏比 (v2.0 新增)** | `avg_W / |avg_L| >= 3.0` | `>= 5.0` |

### 4.5 交易利润结构门槛

| 项目 | 硬门槛 | 优秀线 |
|---|---:|---:|
| `avg_W / |avg_L|` | `>= 1.35` | `>= 1.60` |
| `>3R` 大赢单占比 | `>= 20%` | `>= 50%` |
| `<0.5R` 微利单占比 | `<= 55%` | `<= 40%` |

定义:

- `>3R` 大赢单占比 = `>3R 盈利单数 / 全部盈利单数`
- `<0.5R` 微利单占比 = `<0.5R 盈利单数 / 全部盈利单数`

说明:

- 该组指标直接约束"是否真的让利润奔跑"。
- 如果大赢单占比不足, 说明系统本质仍在切碎趋势利润。

### 4.6 Sharpe / Sortino / Calmar 门槛

| 项目 | 硬门槛 | 优秀线 |
|---|---:|---:|
| Sharpe | `>= 1.5` | `待按品种细化` |
| Sortino | `>= 2.0` | `待按品种细化` |
| Calmar | `>= 1.2` | `待按品种细化` |

注:

- Sharpe / Sortino / Calmar 阈值参考公开通用标准, 当前先作为 WFYS v2.0 风险质量底线。
- 它们只负责风险调整收益质量, 不替代利润能力判断。

### 4.7 待定项

以下项目原则上应纳入 WFYS, 但本轮尚未正式定标, 暂标记为待定:

- 最低样本数
- 最低交易数 (部分纳入 4.1 周均 2 单)
- 分品种的 Sharpe / Sortino / Calmar 优秀线
- BTC / XAU 的总收益率分段

## 5. 100分评分

### 5.1 总体结构

WFYS 评分固定为 100 分:

- `稳定性 30分`
- `利润能力 30分`
- `风险质量 25分` (v2.0 新增周均/胜率/盈亏比)
- `趋势利润结构 15分`

设计原则:

- 稳定性与利润能力等权, 防止滑向 `xau1` 或 `xau2` 的单边极端。
- 风险质量单独计分, 防止净利掩盖路径风险。
- 趋势利润结构独立计分, 防止"微利堆利润"伪装成强策略。

### 5.2 稳定性 30分

| 子项 | 分值 | 说明 |
|---|---:|---|
| 24月盈利月数 | 10 | 至少 22/24 (硬门槛), 23-24 满分 |
| 亏损月控制 | 8 | 亏损月 ≤ 2, 0 亏损月 8 分 |
| 利润集中度 | 12 | 同时考察 `Top 3` 与 `Top 5` |

### 5.3 利润能力 30分

| 子项 | 分值 | 说明 |
|---|---:|---|
| 24独立月总收益能力 | 12 | 主看 `24月总净利 / 初始资金` |
| 720d净利能力 | 10 | 主看 `720d净利 / 初始资金` |
| 强利润月 / 大趋势月数量 | 8 | 防止"只稳不跑" |

### 5.4 风险质量 25分 (v2.0 强化)

| 子项 | 分值 | 说明 |
|---|---:|---|
| 720d回撤 | 8 | Relative Max Drawdown |
| Recovery Factor | 7 | 看回撤后的恢复能力 |
| Profit Factor | 5 | 总体账面健康度 |
| 720d 周均单数 | 2 | v2.0 新增, 主看 `总单数 / 103 周` |
| 720d 胜率 | 2 | v2.0 新增, 主看 `盈利单数 / 总单数` |
| 720d 盈亏比 | 1 | v2.0 新增, 主看 `avg_W / |avg_L|` |

### 5.5 趋势利润结构 15分

| 子项 | 分值 | 说明 |
|---|---:|---|
| `avg_W / |avg_L|` | 5 | 是否放大了右尾 |
| `>3R` 大赢单占比 | 6 | 你的核心"吃满趋势"约束 |
| `<0.5R` 微利单占比 | 4 | 防止过早出场 |

## 6. 分级结论

在全部硬门槛通过后, 再按总分给出固定等级:

| 等级 | 条件 | 结论 |
|---|---|---|
| 淘汰 | 任一硬门槛失败 | 不进入 Live 候选 |
| 继续研究 | 硬门槛通过, 但 `<80分` | 可继续做研究回测 |
| 观察级候选 | `80-84分` | 可列入观察池 |
| 研究版Live候选 | `85-89分` | 可进入研究版 Live 候选 |
| 优先部署级候选 | `>= 90分` | 研究版中优先部署 |

## 8. 自动化脚本 (v2.0)

### 8.1 720d 连续回测

```bash
python scripts/mt5_backtest_win.py --strategy <name> --symbol BTCUSDm \
  --from 2024-06-19 --to 2026-06-07 --model 4 --deposit 200 --timeout 1200
```

输出:
- `results/backtest/<name>_<from>_<to>_<date>.txt` (720d 汇总)
- `results/backtest/<name>_<from>_<to>_<date>.trades.csv` (逐单归因)

### 8.2 24 独立月回测 (v2.0 新增)

```bash
python scripts/run_24m_independent.py --strategy <name> --symbol BTCUSDm --year-start 2024-07
```

跑 24 次独立 30 天回测 (2024-07, 2024-08, ..., 2026-06), 输出:
- `results/backtest/<name>_24m_independent/<name>_<YYYY-MM>.txt` (24 个)
- `results/backtest/<name>_24m_independent_summary.csv` (24 月汇总)
- `results/backtest/<name>_24m_independent_trades.csv` (24 月逐单)

### 8.3 WFYS 评分

```bash
# 24 独立月评分
python scripts/wfys_score.py \
  --monthly-csv <name>_24m_independent_summary.csv \
  --continuous-report <name>_20240619_20260607.txt \
  --trades-csv <name>_24m_independent_trades.csv \
  --spec btc --deposit 200 \
  --json-out <name>_wfys_v2.json
```

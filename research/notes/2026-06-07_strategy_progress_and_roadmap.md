## 三、关键发现

### 3.1 已证实有效

| 功能 | 机制 | 效果量化 |
|:---|:---|:---|
| 噪音门控 | Tick方向+振幅过滤 | 2605秒杀减少60% |

| 自适应Neutral | 防守态拦截无趋势 | +$3-7/月 |
| Mitigation Entry | 消解入场替代Bounce | 2605亏损-45% |
| **双扫确认** | 双方向LP扫荡后才入场 | 天然自适应,全月改善 |

### 3.2 已证伪

| 功能 | 失败原因 |
|:---|:---|
| 防守态冷卻 | M1上交易间隔>200 bars,冷卻无意义 |
| ATR体制检测 | 低波体制不显著, 效果<参数噪声 |
| 方向仓位过滤 | 风险本位lot计算抵消乘数效应 |
| BlockSweepEntry | 杀2605好但杀2505趋势99.6% |
| 时段过滤 | 同一时段不同市场下表现相反 |

### 3.3 结构性限制

2605 无法转盈的根本原因:

```
W/L比值 = avg_W / |avg_L| = $1.07 / $1.72 = 0.62

盈亏平衡所需胜率: WR_req = 1 / (1 + 0.62) = 61.7%
当前PathB+decay 2605胜率:   25.0%
差距:                        36.7pp

要填平这个差距而不破坏2505, 需要将avg_W提升到$5.17(3x改善)
或avg_L降低到$0.36(4.8x改善) — 两者在当前框架下均不可行
```

## 四、下一步改进计划

### 4.1 高优先级 (P0-P1)

#### P0: 生产配置校准 (PathB+decay0.7)

**无需代码改动**。已验证最佳平衡配置:
- 生成正式 .set 文件: `v11xau-qs3-s2-dswp-d07`
- 运行 720d 全量回测(Model 4, Real Ticks)
- 如通过, 可部署为 QS3 的下一个生产版本

```
预期720d: $200→估算 $30K-50K (基于2505+$7.5K和2604-$1的月均推算)
风险: 长期回测可能暴露月度过拟合问题
```

#### P0: 双扫窗口自适应

当前窗口固定 20 bars。可基于 ATR 或近期波动率自动调整:

```
窄幅震荡(低ATR): 窗口拉长 → 更严格的入场条件
趋势扩张(高ATR): 窗口缩短 → 更快响应的入场
```

预期效果: 在2605型市场自动收紧, 2505型市场自动放宽。

#### P1: 防守态DTP=0.5R (H5假说)

在防守态降低止盈目标, 匹配窄幅区间的微利特征:

```
正常态: DTP=1.0R (趋势月跑大利润)
防守态: DTP=0.5R (震荡月抓小利润)
```

**高回报但高风险**: 2505可能因过早止盈而损失大赢。需通过仅防守态启用来控制风险。

### 4.2 中优先级 (P2-P3)

#### P2: Multi-TF 确认叠加

在双扫确认基础上, 加入 M15 周期确认:
- M1 Sweep OB 双扫 → M15 方向一致 → 入场
- 额外多一层趋势保护(2505受益)

#### P3: FVG (公允价值缺口) 交易

SMC 分析的 Path C, 当前代码完全空白的新原语:
- M1 FVG 在震荡市中回补率接近100%
- 每个 FVG 提供 $1-3 微利
- 作为独立收入源, 不与 OB 策略竞争

实现量: ~200行, 包括 FVG 检测 + 入场/退出逻辑。

### 4.3 战略性方向

#### 市场体制自适应 (中长期)

当前最大瓶颈是**无法在趋势/震荡两种体制间有效切换**。建议方向:

```
方案1: 多腿组合 (Portfolio)
  - 腿1: S2+双扫 (趋势月强势)
  - 腿2: 纯OB+DTP=0.5R (震荡月防御)
  - 通过月内权益回撤自动分配权重

方案2: 体制切换 (Regime Switch)
  - 实时检测: ATR趋势/ADX/市场状态
  - 趋势体制 → PathB配置
  - 震荡体制 → 低风险配置

方案3: 多品种分散
  - XAUUSD + 加密货币(BTC/ETH)
  - 不同品种在不同市场体制下表现互补
  - 已有BTC Profile代码基础
```

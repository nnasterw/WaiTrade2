# v10 XAU 策略探索记录

日期：2026-05-18  
窗口：2025.11.19 ~ 2026.05.18，180 天  
品种：XAUUSDm  
回测：MT5 Strategy Tester CLI / Real Ticks / 初始资金 200 USD

## 结论

本轮没有得到满足全部目标的 v10 定版。

目标为：
- 胜率 > 70%
- 平均盈利/平均亏损 > 2
- 日均开单数 > 3

实测结果显示三者存在强冲突：
- 高频 M3 可以达到日均 > 3，但胜率通常 45%~55%，平均盈亏比约 1.0~1.2。
- 早保本/固定 TP 可以把胜率推到 75%+，但平均盈利/亏损只有约 0.3，收益质量不足。
- M5 质量过滤可以得到 PF > 2，但日均只有 0.36~0.48。

因此当前通用 EA 的市价 bounce 入场框架下，单 XAU 单策略暂未验证出满足三指标的 v10。

## 已验证规律

1. 高频与高质量在 XAU 上冲突明显。
   - M3 提供足够交易数，但大量交易中位数仍是 -1R 或 0.08R 小锁盈。
   - M5 可以提高 PF，但交易数严重不足。

2. 胜率可以被早保本制造，但不是长期盈利核心。
   - `V10-M3-IQ50FL` 胜率 76.2%、日均 3.43，但平均盈/亏只有 0.33。
   - 这类高胜率主要来自 0.08R 小赢，无法满足盈亏比 > 2。

3. 盈利核心仍是少数 runner。
   - 不早锁盈的质量策略余额更好，靠右尾拉升账户。
   - 快速固定 TP 会切掉右尾，导致账户持平或亏损。

4. 主动小亏退出不成立。
   - `STRUCTC25/C35` 把中位亏损压到 -0.25/-0.35R，但胜率跌到 19%~24%。
   - 说明很多最终盈利单入场后会先回撤，不能简单用初始逆行判断失败。

5. V8 式结构突破建块有价值，但不足以单独达标。
   - M3 结构突破保留频率，但胜率仍低。
   - M5 结构突破 PF 最高，但日均不足。

6. OB 中线/中下部入场更适合作为加仓逻辑，而不是硬过滤。
   - `V10-M3-DEEP67` 只做深位触及后入场，日均降到约 1.26 且亏损，说明过深过滤会错过大量有效反弹。
   - `V10-M3-TDBOOST` 保留边缘 touch 入场，但若价格触及 OB 中线则加仓，日均 3.33、余额 3433，明显优于结构突破基线。
   - 这支持用户假设：深位若是高质量单，应作为仓位权重，而不是减少基础单数。

7. 有过浮盈后的失败退出比“开仓即小亏退出”更合理。
   - 主动小亏退出 `STRUCTC25/C35` 会砍掉大量先回撤后盈利的单，胜率崩。
   - `TDMFE` 用“曾经浮盈 0.6R，回落至 -0.1R 主动出场”，把平均盈亏比推到 1.92，接近目标 >2；但胜率仍只有 42.1%。
   - 后续应继续研究小周期反转/压力位提损，而不是固定 R 阈值。

8. 现有 M15 顺势过滤和评分过滤不是 XAU 胜率答案。
   - `TDMFE-ST` 胜率只从 42.1% 到 43.5%，日均从 3.69 降到 1.83。
   - `TDMFE-S3` 胜率 43.1%，日均 1.08。
   - 这说明当前 `DetectMarketState`/`CalcSignalScore` 对本轮 XAU 高频结构没有筛出明显高胜率子集。

9. 方向分时过滤有效，但不能解决胜率目标。
   - `V10-M3-TDMFE-DF` 剔除实测亏损簇：买入 1/6/8/10/23 点，卖出 6/16/20/22 点。
   - position 聚合后日均 3.01、PF 1.63、余额 5280，优于 `TDMFE`。
   - 但胜率仍只有 46.4%，说明低胜率不是少数坏时段造成，而是当前入场框架的结构性特征。

10. 大小周期同向 HTF 目标位方向正确，但当前实现没有带来增益。
   - `HTFO` 只启用 HTF 前高/前低或量度目标，不启用强弱转换，余额 5078，低于 DF。
   - 原因推断：当前 M15 swing 目标过近/过早被普通 DTP 和分批逻辑截断，没有真正扩大右尾。
   - 下一步若继续做，应使用“HTF目标单禁用普通DTP早退、只在明确转弱后退出”的专用 runner，而不是简单改 TP。

11. 强弱转换更适合做晚期 runner 管理，不能早用。
   - 用户给出的“强势市场不直接反转，先转弱”规律已落成实验：实体递减、长影线、影线创新高/新低但实体未破、突然反向K，作为动能转弱信号；连续4根以上同向K且回撤小，作为强势信号。
   - `MOM` 将转弱退出延后到 1.8R 后，结果与 DF 基本持平且余额略高。
   - 这说明强弱转换不适合解决入场胜率，但可以作为右尾保护的候选模块继续调参。

12. `RR2` 方向分时能把平均盈亏比推近 2，但代价是日均跌破 3 且净收益下降。
   - 单独清缓存复跑后，position 聚合为日均 2.99、胜率 42.4%、平均盈亏比 1.98、余额 4813。
   - 结论：为了指标删单不如保留 DF/MOM 的净收益质量。

13. Tester 缓存/复用问题再次出现。
   - 批量跑 `RR2/HTFO/MOM` 时，`RR2` 首次报告复用了上一段 `HTF` 结果，日志中没有 `V10-M3-TDMFE-RR2` 加载段。
   - 已单独复跑修正。关键结果必须检查 Agent 日志里的策略加载版本，不能只看报告文件。

14. HTF runner 出场不是当前突破口。
   - `RUN/RUNW/HTFL` 尝试让大小周期同向单跳过或延后普通 DTP/Trail，理论上让利润看向大周期目标。
   - Agent 日志聚合显示最好的 `RUNW` 余额 5387，略高于 `MOM`，但胜率 46.8%、平均盈亏比 1.86，仍未突破指标。
   - 说明问题不是普通 DTP 过早截断，而是当前 HTF 目标识别没有足够区分真正 runner 与普通反弹单。

15. 分批止盈能提高胜率，但会明显牺牲盈亏比。
   - `P05` 全部单 0.5R 先平 35% 并提损，胜率提升到 66.3%，接近目标 70%，但平均盈亏比降到 0.80，余额也下降。
   - `DP` 只对深位单分批，胜率仅 48.8%，平均盈亏比 1.68，弱于 DF/MOM。
   - 结论：分批可以作为资金曲线平滑工具，但不是满足“胜率>70且盈亏比>2”的解。

16. 脚本超时不等于 Agent 无结果。
   - `RUN/P05/DP` 等实验出现 MT5 CLI 报告失败或无数据，但 Agent 日志中存在完整 final balance。
   - 后续长持仓或多次部分平仓策略必须以 Agent 日志段为权威解析，不只看 `results/backtest/*.txt`。

17. bounce 后等待 M1 收盘确认不是有效入场过滤。
   - 新增 `InpBounceCloseConfirmBars` / `InpBounceCloseTF` / `InpBounceCloseBufferPct` / `InpBounceCloseRequireBody`。
   - 单独测试 `C1/C1NB/C2`，以及叠加方向分时的 `DFC1/DFC1NB`，全部低于 `DF`。
   - 最好的是 `DFC1NB`，日均 1.77、胜率 44.5%、平均盈亏比 1.81、余额 2631，明显不如 `DF` 的日均 3.01、胜率 46.4%、平均盈亏比 1.88、余额 5280。
   - 结论：收盘确认主要把好反弹的入场推迟，亏损簇仍然保留；它不适合作为主入场过滤。

18. “无浮盈失败退出”方向合理但无法达标。
   - 新增 `InpNoMFEExitBars` / `InpNoMFEMinPeakR` / `InpNoMFEExitR`。
   - `NM3/NM5/NM5S` 都保住日均 >3，但胜率仅 44.6%~45.4%，平均盈亏比 1.89~1.91，余额 4442~4719。
   - 它确实减少部分坏单，但也误杀先磨损后盈利的单，最终不如 `DF/MOM/RUNW`。
   - 结论：无浮盈过滤可作为风控模块保留，但不能作为 v10 达标核心。

19. 小比例分批 + 提损能制造 70% 胜率，但会破坏盈亏比。
   - `P10L02` 在 0.5R 平 10% 并把余仓 SL 锁到 0.2R，胜率达到 70.6%、日均 3.13。
   - 但平均盈亏比只有 0.65，余额 3673，远低于 DF/MOM/RUNW。
   - `P07L03` 把触发推迟到 0.7R，胜率降到 58.3%，平均盈亏比只有 1.04。
   - 结论：只要早期强提损，大量本可扩展为右尾的单被压成小赢；胜率和盈亏比无法同时达标。

20. 单靠入场过滤无法把高频策略胜率推到 70%。
   - `TDMFE` 约 665 笔、胜率约 43%，若为了日均 >3 至少保留 541 笔，即使删掉的全是亏损单，胜率上限也只有约 53%。
   - `DF/RUNW` 已经只有约 541 笔，无法再通过删单提升到 70% 且保持日均 >3。
   - 因此三目标同时成立需要新的入场范式或多策略/多品种组合，而不是在当前 bounce 市价入场框架上继续删单。

21. 入场前强弱转换过滤能改善盈亏比，但无法解决胜率。
   - 新增 `InpEnableEntryMomentumFilter` / `InpEntryMomentumTF` / `InpEntryBlockCounterStrong` / `InpEntryRequireCounterWeak`。
   - 逻辑：若小周期仍处于反向强势推进，且未出现实体递减、长影线、反向K或影线新高/新低但实体未破，则拒绝逆强势入场。
   - `EFB` 只拦截反向强势未转弱：540 笔，日均 3.00，胜率 45.6%，平均盈亏比 1.93，PF 1.62，余额 5238。
   - `EFW` 强制必须出现反向小周期转弱证据：346 笔，日均 1.92，胜率 43.9%，平均盈亏比 2.11，PF 1.65，余额 3486。
   - `EFRUN` 叠加强势放宽 DTP 和 1.2R 转弱退出：543 笔，日均 3.02，胜率 45.7%，平均盈亏比 1.78，PF 1.49，余额 4398。
   - 结论：用户的“强势不直接反转，先转弱”规律在入场端确实能筛掉一部分低质量逆势单，并把严格版本盈亏比推过 2；但保留日均 >3 时，胜率仍约 45%，无法达标。

22. M5高胜率底座无法通过小亏退出修成高盈亏比。
   - 历史 `R13E` 在 90 天做到日均 3.4、胜率 73%，但平均盈亏比只有约 0.76且余额亏损。
   - `NM2/NM3` 保住 72%~73% 胜率和日均 >3，但平均盈亏比仍只有 0.34~0.35，余额降至 146/156。
   - `C15` 主动 0.15R 小亏退出把平均亏损压到 0.16R，但胜率跌到 32.0%，说明大量最终小赢单开仓后也会先逆行。
   - `MFE` 浮盈后失败退出也只得到 67.1% 胜率、0.43 平均盈亏比。
   - 结论：M5高胜率路线的盈利单主要是早保本/小锁盈，亏损仍接近 -1R；它不具备“胜率>70且盈亏比>2”的结构。

23. 失败退出后反手不是稳定优势。
   - 新增 `InpEnableFailureReverse` 及 `InpReverseOn*` 参数，只在主动失败退出后反向开单，默认关闭。
   - `NM5R` 在 M3 无浮盈失败退出后反手：545 笔，日均 3.03，胜率 45.0%，平均盈亏比 1.87，余额 4299，弱于 `DF/MOM/RUNW` 和 `NM5S`。
   - `C15R` 在 M5 0.15R 主动小亏后反手：1373 笔，日均 7.63，平均盈亏比 2.30，但胜率只有 29.1%，余额 83。
   - 结论：假 OB/失败反弹有时会顺势延续，但反手信号噪音过大；它不能作为达标核心。

24. 入场诊断显示优势特征适合仓位加权，不适合硬过滤。
   - `DFD` 诊断版与 `DF` 同逻辑，541 笔、胜率 45.7%、平均盈亏比 1.94、余额 5280。
   - 卖单明显优于买单：卖单 278 笔、胜率 48.6%、平均盈亏比 2.30、PF 2.17；买单 263 笔、胜率 42.6%、平均盈亏比 1.50、PF 1.11。
   - 连续OB优于非连续OB：251 笔、胜率 47.4%、平均盈亏比 2.19、PF 1.97，但日均只有 1.39。
   - 触碰 100-300 次的 OB 最优：85 笔、胜率 51.8%、平均盈亏比 2.81、PF 3.02，但样本太少。
   - 关闭 DS 权重 `NODS` 小幅提升至余额 5429、PF 1.65、平均盈亏比 1.95；关闭全部乘数余额明显下降到 1671，说明仓位乘数贡献复利，但 DS 权重在 XAU 上有噪音。
   - 结论：下一步更适合做“特征仓位权重/方向差异化风控”，而不是把优势子集硬过滤掉。

25. 方向差异化风控能改善盈亏比，但会明显降低频率，不能解决胜率。
   - 新增 `InpBuy/SellMinStrength`、`InpBuy/SellPosMult`、`InpBuy/SellBE*`、`InpBuy/SellDTP*`，用于分别控制多空质量门槛、仓位和出场。
   - `DIRW` 只调方向仓位：543 笔、日均 3.02、胜率 45.3%、平均盈亏比 1.99、余额 4771，弱于 `NODS`。
   - `DIRF/DIRX` 通过提高买单强度门槛，把平均盈亏比推到 2.18~2.25，但日均只有 1.69，无法满足日均 >3。
   - 结论：卖单优势是真实存在的，卖单 PF 约 2.1~2.3；但买单硬过滤会让频率断层，方向差异化更适合组合权重，不适合作为单策略达标解。

26. 适度放宽并发比强势延续加仓更有效，但仍不能解决胜率。
   - `C3` 只把最大并发从2提高到3：588 笔、日均 3.27、胜率 46.8%、平均盈亏比 2.00、PF 1.75、余额 6719，是当前高频候选里收益质量最好的版本之一。
   - `ADD1/ADDS` 启用“浮盈后小周期强势延续加仓”，总余额约 6700，和 `C3` 基本持平。
   - 真实 ADD 加仓单只有 25 笔，胜率 44.0%、平均盈亏比 1.27、PF 1.00、净R约0，说明加仓本身没有贡献，收益提升主要来自并发从2放到3后释放了更多基础信号。
   - 结论：用户提出的“强势继续持仓吃满利润”在持仓管理上有价值，但当前 4 根连续强势K线加仓条件没有形成高质量加仓信号；短期应保留 C3 作为高频观察候选，而不是把 ADD 作为核心。
   - `C3D` 入场诊断复跑后按真实 position 聚合为 588 笔、日均 3.27、胜率 46.77%、平均盈亏比 1.95、PF 1.71、净R 145.25。CLI 的 deal 配对口径会受部分平仓影响，后续以 position 聚合为准。

27. Bounce确认后短回踩入场被证伪。
   - 新增 `InpEnableConfirmPullback` / `InpConfirmPullbackPct` / `InpConfirmPullbackWaitSec` / `InpConfirmPullbackMaxAdversePct`。
   - 逻辑：触及 OB 后先等待 bounce 确认；确认后不立即追价，而是在 30 秒内等待价格回踩到“触点-确认价”区间的 30% 或 50% 位置，再按真实 tick 市价入场；若反向跌破触点超过 OB 高度 20% 或超时，则放弃。
   - `PB30`：375 笔、日均 2.08、胜率 41.3%、平均盈亏比 1.66、PF 1.17、余额 1293。
   - `PB50`：314 笔、日均 1.74、胜率 43.6%、平均盈亏比 1.72、PF 1.33、余额 1839。
   - 结论：短回踩能改善部分成交价，但会错过大量有效延续单，并且没有过滤掉 2026-05-15 这类连续买单止损簇；频率直接跌破日均 3，不适合作为 v10 核心。该结论也反证：不能简单把“OB中下位置入场”做成硬过滤，更适合只作为仓位加权或加仓标签。

28. 高胜率底座放宽 runner 仍无法把盈亏比推到 2。
   - `P10L02` 的核心是 0.5R 先平 10% 并把余仓 SL 锁到 0.2R：564 笔、日均 3.13、胜率 70.6%、平均盈亏比 0.65、PF 1.55、余额 3673。
   - `P10R3` 在同一底座上把 runner DTP 推迟到 3R，并放宽回撤到 0.35：566 笔、日均 3.14、胜率 70.0%、平均盈亏比 0.66、PF 1.55、余额 3882。
   - `P10R3M` 叠加强弱转换：强势放宽 DTP，1.5R 后转弱止盈。结果 566 笔、日均 3.14、胜率 70.0%、平均盈亏比 0.63、PF 1.46、余额 3745。
   - 分布核心：`P10R3` 有 292 笔盈利落在 0~0.25R，而大亏仍有 167 笔接近 -1R；胜率来自早锁盈，不来自高质量方向判断。
   - 结论：早锁盈可以制造 >70% 胜率，但结构上会把平均盈利压低；放宽 runner 和强弱转换都不能抵消小赢堆积。若仍要求单策略同时满足胜率>70、盈亏比>2、日均>3，当前这套“先小赢锁定 + runner”的出场路线不可行。

29. C3 入场诊断显示：硬过滤无法把胜率推到 70%。
   - `C3D` 共有 588 笔 position，其中盈利 275 笔。若要保住日均 >3，180 天至少保留 541 笔，只能最多删 47 笔。
   - 即使这 47 笔全部都是亏损，理论胜率上限也只有 275 / 541 = 50.83%，远低于 70%。
   - 优势簇存在但样本太少：08 点 34 笔、胜率 79.4%、平均盈亏比 2.83；touch 100-299 共 98 笔、胜率 55.1%、平均盈亏比 3.07；卖单 306 笔、胜率 50.0%、平均盈亏比 2.29。
   - 亏损簇也清晰但不能解决三指标：如 `age=40-79&cont=1&deep=0` 31 笔、胜率 19.4%、净R -16.31；`dir=buy&h1=0&pos_mult=5-6` 26 笔、胜率 26.9%、净R -12.89。
   - 结论：这些特征适合做仓位权重、风险降权、或避开极端坏簇，但无法靠删单把当前高频 OB bounce 框架改造成 70% 胜率。

30. 确认后长等待浅回踩仍然失败。
   - 为避免市价 chase，在 `C3` 底座上测试确认后 120 秒内等待 10%/20%/30% 回踩入场。
   - `PB10W120`：482 笔、日均 2.68、胜率 44.4%、平均盈亏比 1.71、PF 1.37、余额 3264。
   - `PB20W120`：469 笔、日均 2.61、胜率 43.5%、平均盈亏比 1.69、PF 1.30、余额 3098。
   - `PB30W120`：439 笔、日均 2.44、胜率 41.7%、平均盈亏比 1.69、PF 1.21、余额 1754。
   - 结论：等回踩能改善一部分成交价，但错过了更多有效延续单，频率、胜率、PF 全部低于 C3。当前 v10 不应继续沿“确认后回踩市价入场”方向微调。

31. C3 出场路径诊断：胜负分界几乎就是“能否先到 1R”。
   - 新增只影响日志的 `POSITION_GONE_DIAG`，记录持仓关闭前的峰值R、DTP峰值、最后提损来源、是否深位/HTF/反手/加仓。
   - `C3P` 交易结果与 `C3` 完全一致：588 笔、日均 3.27、胜率 46.77%、平均盈亏比 1.95、PF 1.71、余额 6719。
   - 所有盈利单都至少达到 1R；没有任何亏损单曾达到 1R。
   - 亏损单分两类：185 笔从未达到 0.6R，平均 -1.02R；128 笔达到 0.6~0.99R 后触发 MFE fail，平均 -0.123R。
   - 盈利单分两类：117 笔峰值 1.0~1.99R，最终平均 +0.185R；158 笔峰值 >=2R，最终平均 +2.08R，是账户利润来源。
   - 这说明当前出场逻辑已经把 0.6R 后失败的单压成小亏，真正伤害来自“从未给到 0.6R 的满亏单”和“1R 前无法区分慢热赢家/失败单”。

32. 震荡区间有效突破入场已初步证伪，不能直接替代 C3。
   - 按用户建议新增 Range Breakout：把连续 M3 震荡区间视为 OB，上一根K线收盘有效突破后顺势市价入场，SL 放区间另一侧，TP 按区间高度倍数。
   - 严格定义：`RB8/RB12/RB16` 分别使用 8/12/16 根 M3 震荡区间，TP=2 倍区间高度。结果交易数极低：41/12/21 笔，日均 0.1~0.2；`RB12` 小幅盈利到 215，但样本过少。
   - 宽松定义：`RB4L/RB6L/RB8L` 降低突破阈值、放宽区间宽度并取消分时过滤。`RB4L` 达到 638 笔、日均 3.5，但胜率 56.1%、报告盈亏比 1.14、余额 38.86；`RB6L/RB8L` 频率不足且亏损。
   - 纯量度目标：`RB4R` 禁用早保本、MFE fail 和 trail，仅保留 2 倍区间高度 TP。结果 469 笔、日均 2.6、胜率 61.6%、报告盈亏比 1.27、余额 37.74。
   - 结论：XAU M3 的短震荡突破假突破很多；放宽能给频率，但没有正期望；收紧能稍提质量，但频率断层。该方向不应作为 v10 主线，除非后续加入“突破后回踩不破/假突破反向”的二阶段确认。

32. 单纯时间失败退出无法修复满亏单。
   - 从 C3P 的持仓时长看，满亏单中位持仓约 1.95 分钟，0.6~0.99R 回吐小亏中位约 2.85 分钟，赢家到 1R 的中位时间约 2.6~3.1 分钟。
   - 若在 3/5/8 分钟仍未到 1R 就退出，候选集合里亏损占比只有约 49%/48%/50%，并不能稳定区分坏单。
   - 这与 `NM3/NM5/NM5S` 的回测结果一致：无浮盈/超时失败退出能减少部分坏单，但会误伤慢热赢家，无法把胜率推高。
   - 结论：下一步必须在入场前或触及后极早期识别“会不会先到 1R”，而不是继续调 BE、DTP、分批、回踩等待或时间退出。

## 本轮新增 EA 能力

为保持历史策略默认行为，新增参数默认全部禁用：

- `InpMinImpulseBodyPct` / `InpMinImpulseVolRatio`
  - 位移K实体占比和成交量倍率过滤。
- `InpStructureBreakBars` / `InpStructureBreakATR` / `InpRequireImpulseCandleDir`
  - V8 式结构突破建块过滤。
- `InpEntryConfirmBars`
  - bounce 后突破最近 N 根高低点再入场；已验证对 XAU 本轮无效。
- `InpEarlyLossCutR`
  - 入场后未触发保护前主动小亏退出；已验证对 XAU 本轮无效。
- `InpNoBuyHours` / `InpNoSellHours`
  - 方向分时过滤，用于规避方向性亏损簇。
- `InpEnableHTFTarget` 及 `InpHTF*`
  - 大小周期同向时使用 M15 前高/前低或量度目标。
- `InpEnableMomentumRegime` 及 `InpWeak*` / `InpStrong*`
  - 强弱转换持仓管理：强势时放宽 DTP，转弱后退出 runner。
- `InpHTFSkipDTP` / `InpHTFSkipTrail` / `InpHTFDTP*`
  - HTF目标单专属出场控制。实测未改善三指标。
- `InpPartialPostLockR` / `InpPartialOnlyDeep`
  - 部分平仓后提损，以及仅深位单部分平仓。实测胜率和盈亏比冲突明显。
- `InpBounceCloseConfirmBars` / `InpBounceCloseTF` / `InpBounceCloseBufferPct` / `InpBounceCloseRequireBody`
  - bounce 后要求已收盘K线站回OB外侧。实测过滤掉大量有效入场，失败。
- `InpNoMFEExitBars` / `InpNoMFEMinPeakR` / `InpNoMFEExitR`
  - 持仓若在N根K线内没有达到最小浮盈且仍亏损，则主动小亏退出。实测可保频率，但不提高到目标胜率。
- `InpEnableEntryMomentumFilter` / `InpEntryMomentumTF` / `InpEntryBlockCounterStrong` / `InpEntryRequireCounterWeak`
  - 入场前使用小周期强弱转换过滤，避免在对向强势仍未转弱时逆势入场。实测可改善盈亏比，不能同时满足胜率和日均单数。
- `InpEnableFailureReverse` / `InpReverseOnEarlyLoss` / `InpReverseOnMFEFail` / `InpReverseOnNoMFE` / `InpFailureReverse*`
  - 主动失败退出后反手开单，反手单纳入同一持仓管理。实测噪音过大，不适合作为 v10 核心。
- `InpEnableEntryDebug`
  - 打印每笔入场的 OB 年龄、触碰数、强度、供需权重、连续/顺势、H1对齐、risk/ATR、spread/risk、仓位乘数等，用于从真实回测日志提取胜负规律。
- `InpBuy/SellMinStrength` / `InpBuy/SellPosMult` / `InpBuy/SellBE*` / `InpBuy/SellDTP*`
  - 多空方向差异化入场、仓位和出场控制。实测可提高严格过滤版本的盈亏比，但会牺牲日均单数。
- `InpEnableStrongAddOn` / `InpStrongAddOn*`
  - 浮盈后出现小周期强势延续时加仓。实测 ADD 单本身接近打平，当前不作为核心逻辑。
- `InpEnableConfirmPullback` / `InpConfirmPullbackPct` / `InpConfirmPullbackWaitSec` / `InpConfirmPullbackMaxAdversePct`
  - Bounce 确认后等待短回踩再市价入场。实测会显著降低频率和净R，默认禁用。
- `InpEnableRangeBreakout` / `InpRangeBreakout*`
  - 震荡区间有效突破入场：用连续窄幅区间建区，收盘突破后顺势入场，TP 可按区间高度量度。首轮 XAU M3 实测未形成正期望，默认禁用。
- `POSITION_GONE_DIAG` 日志
  - `InpEnableExitDebug=true` 时记录持仓消失前的峰值R、DTP峰值和保护状态，仅用于诊断，不改变交易行为。

## 下一步建议

若继续冲击 v10，不建议继续在当前市价 bounce 出场参数上微调。

更可能的突破方向：

1. 移植完整 V8/V8.4 限价分层入场。
   - 当前 EA 是触及 OB 后 bounce 确认再市价追入，入场价偏差会吃掉盈亏比。
   - V8 核心盈利点是 OB mid/deep/bottom 分层限价成交和更优 R。

2. 移植完整 OB 分类和仓位分级。
   - 当前只加了部分位移质量过滤。
   - V8 分类包含 body、volume、risk/ATR、顺逆势、扩展时段和 fresh/continuation boost。

3. v10 目标可以拆成组合目标，而不是单 XAU 单策略目标。
   - 单 XAU 要日均 > 3 时质量明显下降。
   - 多品种组合更可能在总频率上达标，同时保留单品种质量。

4. 在 `V10-M3-TDMFE` 基础上增加更贴近价格结构的趋势/压力位过滤。
   - 当前它已满足日均 > 3，平均盈亏比接近 2，余额最高。
   - 缺口是胜率；已验证现有 M15 顺势/评分无效。
   - 下一步应实现更具体的压力位逻辑：前高/前低、M5/M15 swing 区、二推不破后提损/反向开单，而不是复用当前粗粒度状态过滤。

5. 下一轮应转向入场端“失败反弹过滤”，而不是继续改出场。
   - 出场端已经验证：早保本、固定TP、分批、HTF runner 都会在胜率/盈亏比之间互相牺牲。
   - 更可能有效的是入场前过滤假 bounce：例如 bounce 后必须站回 OB 外侧并维持 N 个 tick/M1 收盘，或识别“影线假突破、实体未收回”的失败反弹不入场。

6. 若坚持单 XAU 达到三指标，下一步必须做成交路径级特征学习。
   - 当前日志扫描 78 个 MT5 策略段，无一同时满足三项。
   - 已验证的两类极端都失败：`C3` 接近盈亏比/频率但胜率只有约 47%；`P10L02/P10R3` 接近胜率/频率但盈亏比只有约 0.65。
   - 后续应以交易前 N 秒/N 根 M1 的真实路径为样本，区分“触及后直接穿透”“站回 OB 后延续”“二推不破后反转”，再把规律转为入场状态机；继续调 BE/DTP/分批参数边际价值很低。

## 实事求是判断

截至本轮回测，不应将任何当前候选命名为 v10 定版。

当前最可交易的方向仍是质量优先：
- XAU 低频质量候选：`V10-M5-STRUCT` / `V10-M5-IQ55`
- XAU 高频观察候选：`V10-M3-TDMFE-C3`，日均和盈亏比接近目标，余额/PF 当前最好，但胜率只有 46.8%，不能定版。

## 五、4580 OB 真相

```
市场共识: "4580是强压力位, 触及做空赚大钱"
实际数据: 4580区间内27笔, WR=30%, 净亏-$87

真相: 钱不在OB区间内, 而在OB突破后!
  ✅ Sell @4616→4557 (+$59) ← 从OB上方突破追空
  ✅ Sell @4544→4490 (+$53) ← OB下方延续追空
  ✅ Buy  @4478→4546 (+$68) ← 底部反转抄底
```

## 六、改进计划

### P0: Breakout/Breakdown入场 (替代区间内入场)

```
当前: 价格进入OB区间 → 入场 (区间内绞肉)
改进: 价格突破OB区间 → 确认 → 入场 (突破追单)

实现: 在RegisterChannelMonitors中加OB突破检测
  - Sell OB: 价格跌破OB下沿 → 确认做空
  - Buy OB:  价格突破OB上沿 → 确认做多
  - 区间内反弹 → 继续等待, 不做
```

### P1: 方向×趋势强制对齐

```
当前: IsDirectionAlignedWithTrend只是降低仓位(0.4x SL)
改进: H4明确趋势时, 逆势方向完全不入场
  - H4=BULL → 只做Buy, 任何Sell信号丢弃
  - H4=BEAR → 只做Sell, 任何Buy信号丢弃
  - H4=CHOP → 允许双方向, 但收紧重入
```

### P2: 持仓时长自适应

```
当前: TimeExitBars=40 (固定)
改进: 基于市场状态自适应
  - 趋势市(H4=BULL/BEAR): TimeExit=20 (快进快出)
  - 震荡市(H4=CHOP): TimeExit=120+ (给够时间等突破)
  2605数据证明短线在震荡市必死
```

### P3: OB质量×交易频率联动

```
当前: OB Score过滤独立于交易量
改进: 低质量OB=低频交易, 高质量OB=允许重入
  - OB Score>=60 → 允许H4自适应最大重入
  - OB Score 40-60 → 限制重入(最多2次)
  - OB Score<40 → 完全不交易该OB
```

### P4: 区间密度检测 (新增)

```
问题: $4570-4580区间高密度交易+低WR=绞肉机
改进: 统计每个价格区间的近期交易密度和WR
  - 密度>20笔/天 + WR<35% → 标记为"绞肉区"
  - 在绞肉区 → 只允许MTF H4级别信号, 禁M1
```

## 七、BOS Retest 入场（突破回踩）— 2026-06-11 新增

### 7.1 原理

4580 OB 盈利分析发现：钱不在区间内反弹，在 **突破后的回踩追单**。

```
H1 Swing Low @4580 → 跌破(收盘<4580) → BOS熊信号 →
  价格回踩4580(H1级别阻力变支撑) →
  M1 OB在回踩位反弹确认 → 做空 → 宽SL(H1 ATR×0.5) → 追踪结构目标
```

### 7.2 实现

- `DetectSwingBreakInline()`: 每H1 bar检测收盘突破swing high/low
- `SwingBreakSignal`: 存储突破方向/价位/SL，等待回踩
- `RegisterChannelMonitors`: M1 OB在突破位附近确认 → BOS加权入场
- ob_index偏移20000区分BOS信号，`ExecuteChannelConfirmed`特殊处理

### 7.3 与现有三层协同

```
层级           | 职责              | 4580对应
H4/H1 结构     | 趋势方向+目标位    | H4熊BOS→目标4490
H1 Swing Break | 突破检测+回踩等待  | 跌破4580→等回踩
M1 OB          | 精确入场确认       | 回踩位M1 OB反弹
EntryEngine    | 入场执行           | bounce pattern
```

### 7.4 参数

```yaml
bos_retest_entry: true
bos_retest_sl_buffer: 0.5     # SL=0.5×H1_ATR
bos_retest_tolerance: 0.3     # 回踩容忍度(ATR倍数)
bos_retest_max_bars: 480      # BOS信号有效期(M1 bars=8小时)
bos_retest_weight: 1.5        # 仓位权重(×InpMTFH1Weight)
```

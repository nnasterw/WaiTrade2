## 设计理念

V9.6b 的核心思想是「二推代替过滤」：

1. **二推确认入场** → 信号质量更高 → 可以放宽入场过滤(OB宽度、风险/点差)
2. **更早保本(0.2R)** → 接受更多小保本出局 → 换取「不回吐」
3. **更早DTP(1.5R)** → 比v95c的2.0R更积极锁利 → 配合adaptive回撤
4. **无时间退出** → 不强制12bar平仓 → 让追踪止损自然退出
5. **无加仓** → 二推作为入场确认而非加仓触发 → 简化仓位管理

## 入场逻辑

```
1. M1 K线检测OB (bearish candle → bullish impulse ≥ 1.5×ATR, 或反之)
2. 价格首次触及OB区域 (Bid ≤ OB.high 或 Ask ≥ OB.low)
3. 60分钟内二次触及OB (touch_count ≥ 2)
4. 市价入场 (entry = ask/bid, 非OB mid限价)
5. SL = OB edge + 0.1×ATR buffer
6. TP = 0 (DTP模式)
```

## 仓位乘数计算

```
pos_mult = strength(1~5) × fresh(1.5×) × continuation(1.3×) × 1H_boost(3×) × ds_weight(0.5~2.5×)
final_lot = CalcLotSize(2%, risk_price) × pos_mult
```

最大理论乘数: 5 × 1.5 × 1.3 × 3 × 2.5 = **73.1×** → 需保证金检查防爆仓

## 出场逻辑

1. **SL**: entry - risk (或trailing后的current_sl)
2. **保本**: max_r ≥ 0.2R → SL移到 entry + 0.05R
3. **Trail L1**: max_r ≥ 1.0R → SL锁0.2R
4. **Trail L2**: max_r ≥ 2.5R → SL锁 peak × 0.65 (动态推进)
5. **DTP**: max_r ≥ 1.5R → 激活; 回撤 ≥ peak × 0.30 → 平仓

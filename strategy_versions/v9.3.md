# V9.3 — Tick Bounce 20% Confirmation (5min)

发布日期: 2026-05-08

## 关键变动 (vs V9.1)

- 确认阈值: 30% → 20% (tick更敏感，更早确认)
- 超时窗口: 10min → 5min (减少等待)
- 确认方式: 1m bar close → 实时tick bid/ask
- 结果: 交易数恢复至V8.5的97%，胜率60.9%，盈亏比14.31

## 为什么 V9.3 > V8.5

V8.5 回测假设"5m bar内触及OB即成交"，但在tick级别模拟发现：
- 40%信号触及后直接穿越OB(无反弹) → V8.5全部成交并止损(亏0.71R)
- V9.3 只在确认反弹后入场 → 入场价已在有利方向，被止损仅亏0.13R

这就是V9.3净利润是V8.5的2.6倍的原因。

## 核心逻辑

```
5m bar生成V8.4信号
  → 每秒检查实时 bid/ask
    → 价格触及OB入场价(touched)
    → 触及后tick价格反弹 ≥ 20%×OB高度 → 确认，市价入场
    → 5分钟无反弹 → 过期丢弃
```

## 出场逻辑 (不变)

- 0.5R → 保本
- 1.5R → 锁0.5R
- 3.0R+ → 锁max_r×0.65
- 2.0R+回撤35% → 动态止盈
- 80根5m bar → 时间止盈(≥0.5R)

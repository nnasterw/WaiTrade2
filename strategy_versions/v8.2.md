# V8.2 — Limit Order Split Entry

发布日期: 2026-05-07

## 概述

V8.2是策略级突破：取消"5m收盘确认"要求，改为识别OB后直接设限价单。交易数翻倍(+126%)，胜率提升至97.8%，均R提升至2.1。

## 核心改动 vs V8.1

| 维度 | V8.1 | V8.2 |
|------|------|------|
| 入场触发 | 等5m K线收盘+阳线确认 | 价格触及OB即入(限价单) |
| 入场位 | 100% OB中点 | 50%中点 + 50%下2/3 |
| 信号数 | ~19,000/5品种/1.5年 | ~43,000(+126%) |
| 胜率 | 94.1% | 97.8% |
| 均R | 1.5 | 2.1 |

## 入场条件

### OB建仓（与V8.1相同）
- 核心时段(7-21h): body_pct > 0.55, vol_ratio > 0.9
- 扩展时段(22-6h): body_pct > 0.50, vol_ratio > 0.8
- 做多OB: 收盘 > prior_high_30m + 0.10×ATR + 阳线 + 前一根阴线
- 做空OB: 收盘 < prior_low_30m - 0.10×ATR + 阴线 + 前一根阳线
- OB区域 = 前一根K线的 [open, close]
- 动态TTL: 强OB=15根, 弱OB=8根, 默认12根

### 入场方式（V8.2核心改动）

**不等5m收盘确认，OB识别后立即设限价单：**

1. **中点限价 (50%仓位)**:
   - 做多: 在 `OB_mid = (OB.top + OB.bottom) / 2` 设买入限价单
   - 做空: 在 `OB_mid` 设卖出限价单
   - 触发条件: 5m K线的 `low <= OB_mid`（做多）

2. **深入限价 (50%仓位)**:
   - 做多: 在 `OB_deep = OB.bottom + (OB.top - OB.bottom) / 3` 设买入限价单
   - 做空: 在 `OB_deep = OB.top - (OB.top - OB.bottom) / 3` 设卖出限价单
   - 触发条件: 5m K线的 `low <= OB_deep`（做多）
   - 注: 深入位不一定被填充，未填充时该50%仓位不入场

### 为什么取消确认

OB本身就是结构确认：
- 位移K线突破30m结构 = 方向已确认
- OB区域 = 机构订单集中区
- 价格回到OB中点 = 回踩已发生
- 不需要额外的"阳线收盘"确认

等确认的代价：
- 错过126%有效信号（价格触及OB后5m收盘为十字星或微阴线）
- 入场价更差（等收盘时价格已反弹走远）

### 风控过滤（与V8.1相同）
- `spread×1.5 < risk < ATR×3`
- 逆势过滤: 1H趋势反向 且 risk/ATR >= 1.5 → 跳过
- 夜盘19-21h: vol>1.3 + body>0.50 + risk/ATR<1.0
- 扩展22-6h过滤: vol>1.5 + body>0.60 + risk/ATR<0.8 + 顺势

## 止损

`stop = OB.bottom - 0.15×ATR`（V8.1结构止损，不变）

做多: `sl = close[bar-1] - 0.15 × ATR`
做空: `sl = close[bar-1] + 0.15 × ATR + spread`

## 止盈/追踪（与V8.1相同）

多级追踪:
- 0.5R → 锁0.1R
- 0.7R → 锁0.3R
- 1.2R → 锁0.6R
- 1.8R → 锁1.0R
- 2.5R → 锁1.5R
- 4.0R+ → 锁60%峰值

自适应动态止盈: ≥8R回撤18%, ≥4R回撤22%, ≥2.5R回撤28%, 其他32%

高质量/共振单分批止盈: 1.5R出50%，剩余50%宽追踪。

时间止盈: 80根后浮盈>=0.5R即出。

## 仓位分级（与V8.1相同，但每个层级×0.5因为分两笔）

每个信号的`pos_mult`已除以2（因为同一OB会产生mid+deep两笔信号，每笔50%）。

## 冷却
- 同方向单笔亏损后: 冷却6根5m K线
- 同方向连续亏损后: 冷却12根5m K线

## 品种
AUDUSD, EURUSD, GBPUSD, NZDUSD, USDCAD, USDCHF, USDJPY, XAUUSD, EURJPY, GBPJPY

## Live执行要点

在mt5_demo_trading.py中的V8.2 live执行逻辑：

1. **信号生成**: 当检测到新OB时，立即生成两个候选信号(mid+deep)
2. **限价执行**: 设置`entry_tolerance_r=0.10`（比V8的0.35更紧），要求当前价格非常接近限价位
3. **深入位**: 如果当前bid/ask已超过deep位，deep信号仍等待价格回来
4. **不需确认**: 不等5m K线收盘，每秒检查价格是否在限价位附近
5. **标记**: `use_limit_order=True`，live runner可据此使用MT5 limit order而非market order

### v99g2 — 贵金属+外汇候选 (DTP runner)

**定位**: 在 v99g1 入场纪律不变的前提下，只改大赢出场方式。v99g1 的 DTP 在首次回撤时全平，确定性强，但可能截断 3R+ 右尾；v99g2 改成首次 DTP 回撤先落袋 50%，剩余仓锁到 1R，并从当前 R 重新累计余仓 DTP 峰值。

```yaml
v99g2:
  <<: v99g1
  dtp_exit_mode: 1                 # 首次DTP回撤只平部分
  dtp_partial_pct: 50              # 先落袋50%
  dtp_post_partial_retrace: 0.30   # 余仓给30%回撤空间
  dtp_post_partial_lock_r: 1.0     # 可行时把余仓SL抬到1R
  dtp_reset_peak_after_partial: true # 余仓从当前R重新计算峰值
```

**验证状态**: 已完成 EA/参数/preset 实现。`v99g3` 已改为 XAU 专属 C2 Guard 候选，不再作为“不重置峰值”对照组。

### v99g3 — XAU 候选 (C2 Guard)

**定位**: 将 `xau_hv_c2_guard` 提升为正式 `v99g3`。它继承 v99g2 的 DTP runner，并加入更贴近 live 的守护参数：禁止高风险小时入场、限制仓位乘数和最大手数、交易请求失败后冷却，同时允许 XAU 并发 2 仓。

```yaml
v99g3:
  <<: v99g2
  max_concurrent: 2
  no_entry_hours: "0,9,12,17,18"
  max_pos_mult: 8.0
  max_lot_size: 0.08
  close_retry_cooldown_sec: 60
```

**XAU 180天验证**: `2025.11.19 ~ 2026.05.18`，MT5 Strategy Tester CLI / Real Ticks / 初始资金 `$200`。`xau_hv_c2_guard` 结果为 306 笔、日均 1.7、WR 53.9%、PF 1.11、余额 `$3474.29`。它提高收益弹性，但 PF/WR 低于 `xau_g2_p0_guard`，后续 live 观察应重点看回撤和连亏簇。

## 8-Gap 修复 (内置于EA代码)

所有 v99 系列策略都包含这些代码级修复:

1. **OB实体定义**: K线实体(open/close)，不含影线
2. **Displacement突破**: 必须突破前3根K线高/低点
3. **K线质量**: body/range >= 50%
4. **夜盘过滤**: 23:00-06:00 不创建OB
5. **信号分级**: strength < 0.5 不入场
6. **动态TTL**: 高quality +50%寿命, 低quality -50%
7. **Risk上限**: risk > 3×ATR 不入场
8. **逆势限制**: 逆势 + risk > 1.5ATR 不入场

2026-05-18 起，Gap3/GAP4/GAP5/GAP7/GAP8 已参数化，默认值保持上述行为不变:

```yaml
min_ob_body_pct: 50.0
no_ob_start_hour: 23
no_ob_end_hour: 6
min_ob_strength: 0.5
max_risk_atr: 3.0
max_counter_risk_atr: 1.5
```

## EntryEngine 状态机

```
PHASE_WAITING_TOUCH → PHASE_WAITING_BOUNCE → 确认入场

1. 新bar时注册活跃OB为monitor
2. 每tick检查价格是否触及OB区间
3. 触及后等待bounce确认(ob_height×40%反弹)
4. bounce达标后offset guard检查(偏移<0.5R)
5. 通过→生成TradeSignal→ExecuteSignal
```

## 入场逻辑 (confirm_price 直接入场)

```
1. 1m K线上识别 OB (generate_ob_signals_v84)
2. 价格触及 OB → bounce ≥ 30%×OB高度
3. 直接用 confirm_price 入场 (不等30s, 不用限价单)
4. offset guard: confirm_price vs entry > 1R → 拒绝
```

### 为什么不用限价单/post_confirm

- **限价单**: 精确回调到entry的交易大多是方向反转 → WR 19%
- **post_confirm 30s**: 让价格跑远再追 → live offset 20R+
- **confirm_price入场**: bounce瞬间价格在entry+0.3R → 天然接近

## 出场逻辑

1. SL: entry - risk (或被trailing移动后的current_sl)
2. 保本: max_r ≥ 0.5R → SL移到 entry + 0.1R (覆盖点差)
3. Trailing: max_r ≥ 1.5R → 锁0.3R; max_r ≥ 3.0R → 锁 peak×0.65
4. DTP: max_r ≥ 2.0R 且回撤 ≥ 35% → 平仓
5. Time TP: ≥12bars且pnl>0.5R → 平仓
6. Time decay: 50%时间后收紧, 75%后>0.5R即平
7. Timeout: 12bars后无论如何平仓

## 加仓逻辑 (dt_addon)

- 入场后90分钟内如果OB被再次触及+bounce确认 → 独立开第二笔交易
- 加仓入场价 = 二推确认时的价格 (非原始entry)
- 加仓仓位 = dt_addon_boost × 基础仓位
- 独立止损/trailing/记录

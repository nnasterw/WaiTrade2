# V9.5d — 宽松出场 + 高位紧保护 (A路线极致收益)

## 来源

基于V9.5c的3轮对比实验 (XAU 30天逐tick, 共50+变体)，A路线冠军:

| 因子 | V9.5c | V9.5d | 实验ΔnetR |
|------|-------|-------|-----------|
| 信号过滤 | OB≥3×spread, risk≥5×spread | OB≥2×spread, risk≥3×spread | +59.7 |
| Trailing | 0.5R保本, 1.5R/3.0R | 0.3R保本, 0.8R/1.5R/3.0R紧阶梯 | +33.4 |
| Bounce确认 | 30% | 40% | +24.7 |
| DTP止盈 | 2.0R触发35%回撤 | 禁用 (纯trailing出场) | +24.2 |
| 持仓时间 | 12bars (12min) | 24bars (24min) | +19.1 |

## 核心配方

```yaml
v95d:
  bar_period_min: 1
  bars: 5000
  bounce_pct: 0.40           # ↑ 30%→40% 更严格确认
  timeout_min: 90
  entry_mode: "confirm"
  max_entry_offset_r: 1.0
  min_ob_spread_mult: 2.0    # ↓ 3→2 更多交易机会
  min_risk_spread_ratio: 3.0 # ↓ 5→3 更多交易机会
  breakeven_r: 0.3           # ↓ 0.5→0.3 更早保本
  breakeven_lock_r: 0.05
  trail_levels:
    - {trigger_r: 0.8, lock_r: 0.1}   # 新增: 0.8R锁0.1R
```

## 设计逻辑

### 为什么宽过滤 (+59.7R)
- V9.5c的min_ob_spread_mult=3, min_risk_spread_ratio=5过滤掉了大量可盈利信号
- 放宽到2/3后交易量从1835→4583 (+150%), WR仅降2.3%
- 量的补偿远大于质的损失

### 为什么紧trailing (+33.4R)
- 0.3R即保本 → 快速保护利润
- 0.8R开始锁利 → 减少回吐
- PF从1.50→2.01, 盈亏比大幅提升
- WR降5%但avg_loss从0.77→0.57, 亏损更小

### 为什么高bounce (+24.7R)
- 30%→40%: 更严格的反弹确认 → 减少假突破入场
- 交易减少90笔但WR提升1.1%, 净效果正向

### 为什么禁用DTP (+24.2R)
- DTP在2.0R触发时过早截断利润
- 紧trailing已经能保护利润, DTP多余
- 让大盈利单自然走trailing出场

### 为什么24bars (+19.1R)
- 12min持仓太短, 趋势未充分展开
- 24min给更多时间, WR基本不变但avg_win提升

## 风险点

1. **filter_loose风险**: 更多小OB信号 → 可能在某些品种(低spread外汇)产生过多噪音交易
2. **无DTP风险**: 极端行情中大盈利单可能全部回吐(仅靠trailing)
3. **紧trailing风险**: 在高波动品种可能过早被震出
4. **组合叠加**: 各因子单独测试有效, 组合后可能有交互效应
